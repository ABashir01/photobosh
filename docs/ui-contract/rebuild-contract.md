# Rebuild Contract

- Rebuild instead of patching if the structure conflicts with the approved flow.
- Preserve behavior only where it serves the new screen model.
- Prefer deleting obsolete UI surfaces rather than hiding them under new styling.

## Hard Structural Contract

- One visible phase-specific screen at a time.
- Capture uses the same booth view, but with immersive dark treatment.
- Theme selection must always expose a working download path without a separate finalize step.

## Review Questions

- Does the product still read like a dashboard anywhere?
- Is mobile clearly the first-class composition?
- During capture, is anything visible besides the booth and countdown?
- During theme selection, can the user switch themes and download immediately?
